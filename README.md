# Bond Strategy: Simple Trend Cash

This repository contains the notebook and data for the Simple Trend Cash U.S. Treasury strategy research.

## Contents

- `notebook/06_simple_trend_cash_comparison.ipynb`
  - Final proposal notebook for Miyazaki-style relative-value signals, Simple Trend Cash, BEI variants, and `MZ_TREND_CASH_ADVANCED_V2`. It reads only `data/us_treasury_macro_for_06.xlsx` as its market-data input.
- `notebook/01_data_loading.ipynb`
  - Earlier FRED data-loading notebook kept for reference.

- `notebook/03_us_treasury_monthly_strategy_2006_2026.ipynb`
  - Long-history notebook that fetches/generates the 2004-2026 Treasury yield dataset and runs the earlier monthly strategy backtest.
- `notebook/04_miyazaki_long_comparison.ipynb`
  - Long-history comparison notebook for Miyazaki-style variants.
- `notebook/05_defensive_cash_gated_strategy.ipynb`
  - Earlier defensive cash-gated strategy notebook kept as a reference comparison.

  - Small helper functions used by the earlier monthly strategy notebook.

  - Earlier monthly U.S. Treasury yield curve workbook kept for reference notebooks.

- `data/us_macro_rates_2004_2026.xlsx`
  - Earlier monthly macro/rate workbook kept for reference notebooks.
- `data/us_treasury_macro_for_06.xlsx`
  - Primary input workbook for `06_simple_trend_cash_comparison.ipynb`. It contains 1981-2026 Treasury curve data, macro/rate data, coverage checks, missing-data checks, and metadata.

## Main Strategy

The main proposal remains `MZ_SIMPLE_TREND_CASH`.

It starts from the Miyazaki-style signal, converted to monthly comparison by `signal / 6`, and subtracts a transparent duration-loss penalty based on recent 2Y and 10Y yield momentum. Cash is included directly in the same optimization problem.

`MZ_TREND_CASH_ADVANCED_V2` is included as an improvement candidate. It adds a policy-inflation defense condition and a 2s10 inversion cash hurdle, aiming to preserve more return in low-inflation rally/range markets while still defending against 2022-style tightening shocks.

## Reproduce

To rerun the final Simple Trend Cash flow from data acquisition to backtest:

```bash
uv run jupyter nbconvert --to notebook --execute --inplace notebook/00_data_acquisition_for_06.ipynb
uv run jupyter nbconvert --to notebook --execute --inplace notebook/06_simple_trend_cash_comparison.ipynb
```

The notebooks write:

```text
data/us_treasury_macro_for_06.xlsx
data/backtest_simple_trend_cash_comparison.xlsx
```

The current 06 input design deliberately treats BEI/TIPS as optional: `T10YIE` and `DFII10` only start in 2003, so they do not constrain the core 1981-start backtest. The final proposal `MZ_SIMPLE_TREND_CASH` does not require BEI.

To rerun the long-history data fetch / earlier strategy notebook:

```bash
uv run jupyter nbconvert --to notebook --execute --inplace notebook/03_us_treasury_monthly_strategy_2006_2026.ipynb
```

To rerun all checked notebooks:

```bash
for nb in notebook/*.ipynb; do
  uv run jupyter nbconvert --to notebook --execute --inplace "$nb"
done
```

The FRED API key should be supplied through `.private/.env` as:

```text
FRED_API_KEY="..."
```
